## Completed Task: Create internal/config/config.go

### What was implemented:

1. **Created directory structure**: `internal/config/`

2. **Implemented config.go with the following components**:
   - `AuthConfig` struct: Holds access_token, refresh_token, and expires_at fields
   - `DefaultsConfig` struct: Holds address_id, payment_id, and output_format fields
   - `RateLimitConfig` struct: Holds min_delay_ms, max_delay_ms, and max_retries fields
   - `Config` struct: Main configuration structure combining all sub-configs
   - `DefaultConfig()` function: Returns sensible defaults (output_format: "json", rate limiting: 1000-5000ms, max retries: 3)
   - `GetConfigPath(customPath string)` function: Returns custom path or default ~/.amazon-cli/config.json
   - `LoadConfig(path string)` function: Loads config from JSON file, returns default config if file doesn't exist (first run scenario)
   - `SaveConfig(config *Config, path string)` function: Saves config to JSON file with proper permissions (directory: 0700, file: 0600)

3. **Created comprehensive unit tests** (config_test.go):
   - TestDefaultConfig: Verifies default configuration values
   - TestGetConfigPath: Tests custom path and default path logic
   - TestLoadConfig_NonExistent: Ensures graceful handling of missing config file
   - TestSaveAndLoadConfig: Tests full save/load roundtrip with verification of all fields and permissions
   - TestLoadConfig_InvalidJSON: Tests error handling for malformed JSON
   - TestJSONMarshaling: Verifies JSON serialization/deserialization
