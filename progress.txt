## Implementation: Orders History Command

### What was implemented:
Implemented the `amazon-cli orders history` command as specified in the PRD.

### Files created:
1. **main.go** - CLI entry point
2. **cmd/root.go** - Root Cobra command with global flags and error handling
3. **cmd/orders.go** - Orders command with history subcommand
4. **cmd/orders_test.go** - Tests for orders history command logic
5. **pkg/models/order.go** - Order data models (Order, OrderItem, Tracking, OrdersResponse)
6. **pkg/models/errors.go** - Error handling models and constants
7. **internal/amazon/client.go** - Amazon HTTP client infrastructure
8. **internal/amazon/orders.go** - Order-related API methods including GetOrderHistory
9. **internal/amazon/orders_test.go** - Comprehensive tests for order history functionality

### Project initialization:
- Initialized Go module: `github.com/zkwentz/amazon-cli`
- Created directory structure (cmd, internal/amazon, pkg/models)
- Installed Cobra CLI framework dependency

### Command features:
- Command: `amazon-cli orders history [--year YYYY]`
- Defaults to current year if --year not specified
- Validates year range (1995 to current year)
- Returns structured JSON output matching PRD schema
- Includes verbose logging with --verbose flag
- Proper error handling with appropriate error codes

### Testing:

- Year validation tests (too old, too new, edge cases)
- Response structure validation
