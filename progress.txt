## Task: Test ASIN Validation

### Implementation Summary

Created comprehensive ASIN (Amazon Standard Identification Number) validation functionality with extensive test coverage.

### Files Created

1. **internal/validation/asin.go**
   - Implemented `ValidateASIN(asin string) error` function that validates ASIN format
   - Implemented `IsValidASIN(asin string) bool` convenience function
   - ASIN validation rules:
     - Must be exactly 10 characters long
     - Must contain only uppercase letters (A-Z) and digits (0-9)
     - Cannot be empty
   - Uses compiled regex for efficient validation: `^[A-Z0-9]{10}$`

2. **internal/validation/asin_test.go**
   - Created 17 test cases for `ValidateASIN()` covering:
     - Valid ASINs (letters only, numbers only, mixed, various positions)
     - Invalid ASINs (empty, too short, too long, lowercase, special chars, spaces, mixed case)
   - Created 9 test cases for `IsValidASIN()` boolean helper
   - Added benchmark tests for performance validation
   - All tests pass successfully

3. **go.mod**
   - Initialized Go module: `github.com/zkwentz/amazon-cli`

### Example Usage

```go
import "github.com/zkwentz/amazon-cli/internal/validation"

// Method 1: Error handling
if err := validation.ValidateASIN("B08N5WRWNW"); err != nil {
    return fmt.Errorf("invalid ASIN: %w", err)
}

// Method 2: Boolean check
if !validation.IsValidASIN("B08N5WRWNW") {
    return errors.New("invalid ASIN format")
}
```
